
# Mock-Data 模拟数据生成系统

---

## 一、系统功能概述

这是一个**移动App用户行为模拟数据生成系统**，用于模拟真实用户在App中的各种操作行为，生成符合真实场景的日志数据，可用于：
- **Flink流计算平台**的数据源测试
- **数据分析系统**的功能验证
- **服务端接口**的压力测试
- **日志采集系统**的接收测试

### 核心能力
1. **多页面路径模拟**：模拟用户从启动App到完成支付的完整业务流程
2. **多种用户行为**：启动、浏览、点击、搜索、加购、下单、支付等
3. **曝光/错误数据**：页面曝光记录、错误异常记录
4. **灵活配置**：通过配置文件控制数据规模、行为概率、发送方式

## 四、核心模块详解

### 4.1 配置模块 (AppConfig)

**职责**：统一管理所有可配置参数

```properties
# 配置示例
mock.type=http                    # 输出模式（log/http）
mock.url=http://localhost:8080    # HTTP目标地址
mock.startup.count=10000          # 模拟次数
mock.max.uid=500                   # 最大用户数
mock.error.rate=3                  # 错误率(%)
mock.page.during-time-ms=20000    # 页面停留时间
```

### 4.2 模拟核心 (Mocker)

**职责**：根据用户路径配置，生成完整的用户行为日志

```
数据生成流程：
┌────────────────────────────────────────────────────────────┐
│  1. 初始化                                                    │
│     ├── 获取基准时间戳                                         │
│     └── 构建AppCommon（设备/用户信息）                           │
└────────────────────────────────────────────────────────────┘
                           ↓
┌────────────────────────────────────────────────────────────┐
│  2. 生成启动日志 (AppStart)                                  │
│     └── loading_time（启动耗时）                              │
└────────────────────────────────────────────────────────────┘
                           ↓
┌────────────────────────────────────────────────────────────┐
│  3. 读取访问路径配置 (path.json)                             │
│     └── 根据rate权重随机选择一条路径                           │
│                                                             │
│     路径示例：                                                │
│     home → good_list → good_detail → cart → trade → payment │
└────────────────────────────────────────────────────────────┘
                           ↓
┌────────────────────────────────────────────────────────────┐
│  4. 遍历路径生成各页面日志                                     │
│     ├── AppPage（页面信息）                                   │
│     ├── AppAction（用户动作列表）                             │
│     ├── AppDisplay（曝光记录列表）                             │
│     └── AppError（随机错误，概率3%）                           │
└────────────────────────────────────────────────────────────┘
                           ↓
┌────────────────────────────────────────────────────────────┐
│  5. 输出数据                                                  │
│     ├── mock.type=log  → 输出到日志文件                       │
│     └── mock.type=http → POST到服务端                        │
└────────────────────────────────────────────────────────────┘
```

### 4.3 数据模型 (Bean)

| 模型类 | 字段说明 | 对应真实场景 |
|--------|---------|-------------|
| **AppCommon** | mid, uid, os, ver, channel | 设备信息、用户ID |
| **AppStart** | loading_time, network | 启动耗时、网络类型 |
| **AppPage** | page_id, refer_page, during_time | 页面访问、来源、停留时长 |
| **AppAction** | action_id, item_id, item_type | 点击、加购等动作 |
| **AppDisplay** | display_type, item_id | 商品曝光记录 |
| **AppError** | error_type, error_msg | 异常错误信息 |

## 五、数据输出模式

### 5.1 日志模式 (mock.type=log)

```java
// 直接输出到日志文件
log.info(appMain.toString());
// 或
System.out.println(appMain.toString());
```

### 5.2 HTTP模式 (mock.type=http)

```java
// POST到HTTP服务端
HttpUtil.post(appMain.toString());

// 发送示例：
POST http://localhost:8080/applog
Content-Type: application/json

{
  "common": {"mid": "xxx", "uid": "xxx"},
  "page": {...},
  "actions": [...],
  "displays": [...],
  "ts": 1719999999999
}
```

## 七、使用流程

```
1. 修改 application.properties
   ├── 设置 mock.type (log/http)
   ├── 设置 mock.url (HTTP模式目标地址)
   ├── 设置 mock.startup.count (模拟次数)
   └── 调整其他参数

2. 修改 path.json (可选)
   └── 自定义用户访问路径和权重

3. 编译打包
   mvn clean package

4. 运行
   java -jar mock-data-0.0.1-SNAPSHOT.jar

5. 查看结果
   ├── 日志模式：查看控制台或日志文件
   └── HTTP模式：服务端接收数据
```
